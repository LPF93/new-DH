# 数据采集系统当前架构说明

本文档描述当前仓库已经落地的主架构，不再包含早期演示模式设计。

## 目标

- 通过设备 SDK 回调实时采集数据
- 在 UI 中完成初始化、通道管理、采样状态展示和结果预览
- 将采样数据直接写入用户指定目录下的 TDMS 文件
- 保障命名规则清晰、编码统一、界面简洁，符合企业软件使用习惯

## 当前主链路

1. `AcqShell.UI` 负责配置目录选择、设备列表展示、通道启停和存储控制
2. `AcqEngine.DeviceSdk` 负责设备 SDK 初始化、回调接收和数据块封装
3. `AcqEngine.Core` 负责采集块、会话上下文和公共模型
4. `AcqEngine.Storage.Abstractions` 负责存储编排、命名策略和输出抽象
5. `AcqEngine.Storage.Tdms` 负责 TDMS 文件创建和写入
6. `AcqEngine.Host` 负责独立进程模式下的采集调度

## 存储约束

- 存储目录由用户直接指定
- 文件直接生成在目标目录下，不再创建多级会话目录
- 默认只生成一个 TDMS 文件，不再做分段轮转
- 文件名支持自定义和按开始时间命名
- 同名文件按递增后缀处理

## UI 约束

- 数据源只保留设备 SDK 模式
- 存储配置只保留必要业务项
- 删除旧演示模式、格式说明、会话状态等历史调试型界面元素
- 所有中文文案统一为 UTF-8
